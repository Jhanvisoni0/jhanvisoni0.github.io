Real Assets Investment Intelligence Platform

Source Code ↗

An end-to-end investment analytics platform that simulates what a Real Assets desk analyst does at an institutional investment firm — tracking 15 real asset investments across 6 years of market data, computing risk metrics, detecting market anomalies automatically, and forecasting future returns, all delivered through an executive dashboard.

The Problem

Investment firms managing Real Assets portfolios — real estate, infrastructure, commodities — face a common challenge: price data alone doesn't tell the full story. Analysts spend hours manually pulling data from multiple sources, calculating risk metrics in Excel, and trying to spot underperforming assets before they become a problem. There is no unified system that automatically ingests live market data, computes portfolio-wide risk metrics, flags anomalous behavior, and delivers everything in one executive dashboard — updated and ready every morning.

The Solution

A fully automated, cloud-native investment analytics platform built on Databricks that:

  • Pulls live market data for 15 real assets daily from Yahoo Finance and Federal Reserve APIs
  • Processes and enriches data through a Bronze → Silver → Gold pipeline (Medallion Architecture)
  • Computes 8 financial metrics per asset per day — returns, volatility, Sharpe ratio, drawdown, and benchmark correlation
  • Detects market stress events automatically using an Isolation Forest ML model
  • Forecasts 90-day forward returns using Prophet time series modeling
  • Delivers all insights through a live executive dashboard — no manual work required

How It Works — Pipeline Flow

Extract

STEP 01

Yahoo Finance API → 22,635 rows of daily OHLCV prices (15 assets, 6 years). FRED API → 3,219 rows of macroeconomic indicators (interest rates, CPI, oil).

Load — Bronze Layer

STEP 02

Raw data written as-is to Delta Lake on Unity Catalog Volumes, partitioned by ticker for fast downstream reads, schema enforced via PySpark StructType.

Transform — Silver Layer

STEP 03

PySpark Window functions compute daily & log returns, 30/90-day rolling volatility, Sharpe ratio, maximum drawdown, and rolling 90-day correlation vs. the S&P 500 benchmark.

Aggregate — Gold Layer

STEP 04

Portfolio summary metrics per asset, NAV tracking for a simulated $10,000 portfolio (2019–2024), z-score anomaly pre-flags, and a Fed Funds Rate vs. REIT performance join.

ML Models

STEP 05

Isolation Forest for unsupervised anomaly detection (1,131 stress events flagged) and Prophet for 90-day return forecasting with confidence intervals — both tracked in MLflow.

Deliver

RESULT

A live Databricks SQL dashboard with 4 panels: portfolio NAV over time, asset returns, anomaly timeline, and return forecast.

Real Case Scenario

A portfolio manager at a real assets investment firm opens their dashboard the morning after the crash bottom. The anomaly detection model has already flagged 23 assets across the universe as anomalous — Sharpe ratios have collapsed to -10, volatility has spiked to 57%, and USO (oil) has dropped 84% from its peak.

Key Design Decisions

01

ELT over ETL

Raw data is loaded to Bronze as-is before any transformation. If business logic changes, we reprocess from Bronze without re-calling the APIs — traditional ETL would require re-ingestion every time.

02

Medallion Architecture

Bronze = raw, Silver = enriched, Gold = business-ready. Each layer serves a different consumer, making the pipeline maintainable and scalable independently at each stage.

03

Delta Lake over Parquet or CSV

ACID transactions, time travel, and schema enforcement that CSV and Parquet can't provide. A failed write rolls back automatically; "what did the portfolio look like on March 23, 2020?" becomes a single SQL query.

04

Partitioning by ticker

All downstream queries filter by ticker, so partitioning reduces scan size from 22,635 rows to 1,509 rows for a single-asset query — a 15x performance improvement at this scale.

05

Unsupervised ML for anomaly detection

No labeled training data exists for "this day was anomalous." Isolation Forest discovers anomalies by finding points hardest to explain using the normal feature distribution — no human labeling required.

06

MLflow for experiment tracking

Every model run logs parameters, metrics, and the trained artifact — results are reproducible, model versions are auditable, and the path to production is governed.

The Foundation of Our Pipeline

  • Durability — Delta Lake ensures no data is ever lost or partially written. ACID transactions guarantee every write is complete or rolled back, surviving cluster failures, session timeouts, and mid-write crashes.
  • Idempotency — Every notebook can be rerun safely. .mode("overwrite") ensures running the pipeline 10 times produces the same result as running it once — no duplicates, no corruption on reruns.
  • Separation of concerns — Each notebook has one job. Ingestion doesn't transform. Transformation doesn't aggregate. Aggregation doesn't model. Each layer stays independently testable, debuggable, and replaceable.

Benefits of This Architecture

  • Zero manual data collection — the traditional approach (downloading CSVs, combining in Excel) takes 2-3 hours daily. This platform does it in 60 seconds via API, fully automated.
  • Proactive anomaly detection — instead of discovering underperforming assets during monthly reviews, Isolation Forest flags stress events automatically as they happen — 1,131 anomalies detected across the 6-year window, available the same day.
  • Single source of truth — analysts, portfolio managers, and risk teams all access the same Gold layer tables. The Delta Lake transaction log ensures everyone reads the same committed data simultaneously.
  • Macro-driven insight, not just price data — joining Fed Funds Rate data with REIT returns revealed REITs return 23.26% annually in low-rate environments but -8.17% during rate transitions — a nuance invisible in price data alone.
  • Reproducible ML — every model run is logged in MLflow. Six months from now, you can reproduce exactly what the model did on any given day — non-optional auditability in a regulated environment like finance.

Large Scale Scenario

Imagine this same platform at a firm like MSCI or BlackRock managing not 15 assets but 50,000 — covering every listed REIT, infrastructure company, and commodity ETF globally across 20 years of daily data.

The architecture doesn't change. The Medallion pattern, Delta Lake, PySpark, and MLflow scale from 15 assets to 50,000 without redesigning the foundation.

At that scale:

  • Bronze would hold 365 million+ rows of price data
  • PySpark's distributed processing would split the computation across hundreds of nodes — what takes 60 seconds for 15 assets takes minutes, not days, for 50,000
  • Partitioning by ticker becomes critical — a query for VNQ scans 1 folder out of 50,000 instead of the entire dataset
  • Delta Lake's ACID guarantees become essential — concurrent writes from multiple ingestion jobs happen simultaneously without corrupting each other
  • MLflow's model registry manages hundreds of asset-specific forecast models — each registered, versioned, and promoted through Staging → Production independently
  • The Gold layer feeds dozens of downstream consumers — risk systems, client reporting engines, trading desks — all reading from the same governed Unity Catalog tables
  • Databricks Autoloader replaces the manual ingestion pattern, detecting new data files automatically and processing them incrementally without full reruns

That's the power of building on enterprise-grade tools from day one.
